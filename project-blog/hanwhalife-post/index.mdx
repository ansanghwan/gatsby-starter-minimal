- ### 🗓️ 프로젝트 기간

  2021.04.15 ~ 2021.06.30

- ### 📌 프로젝트 개요

  한화생명 App(4월 오픈) 내 **7월 오픈 예정 ‘구독 보험’** 화면 퍼블리싱을 지원하기 위해 투입.  
  산출물 전달 방식(기존 방식 vs HTML/CSS 분리)에 대한 이견을 **실무 파트(개발/퍼블/디자인)**와 재정의하여 통일.

- ### 🛠 주요 업무

  - 실무자 협의로 **styled-components 기반 마크업** 정책 확정 (_TypeScript 제외_)
  - **전역에서 재사용 가능한 styled-components 구성** 및 공통 UI 컴포넌트 설계
  - **BEM 철학을 반영한 컴포넌트 네이밍**으로 가독성과 유지보수성 강화
  - 요청 화면 퍼블리싱 및 **컴포넌트 재사용성 검증**
  - 산출물 구조/가이드 정리 및 **인수인계 문서화**

- ### 🎯 성과 및 배운 점

  - 전달 방식 혼선을 조정해 **퍼블리싱-개발 협업 표준**을 확립
  - styled-components의 스코프 특성에 **BEM식 역할/상태 네이밍**을 결합해 **코드 일관성과 재사용성 향상**
  - 공통 컴포넌트 가이드/리스트 제공으로 **후속 작업 생산성 향상**
  - React 환경에서의 **컴포넌트화·재사용성 중심 퍼블리싱 역량** 강화
